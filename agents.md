# Project Instructions
## Code Style

- Maximize split code
- Maximize code re-usage
- Create utils, helpers, constants in corresponded folders.
- Always split big chunk of code, starting from 300 lines.
- Explicitly declare all types
- Avoid using any type
  
## Architecture
- Follow the repository pattern
- Keep business logic in service layers
- Run Linter and build after each task
- Update documentation after updating logic
- move logic into hooks, do not overgrow components
